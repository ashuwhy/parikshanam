Architecture Approach

How should the admin and teacher portals be structured in the monorepo?

A

Two Separate Apps

apps/admin → admin.parikshanam.com
apps/teacher → teach.parikshanam.com

Each is its own Next.js app. Shared UI lives in packages/ui. Independent deploys, clean separation.

Pros

  • Clear role separation
  • Independent deployments
  • Easier to scale teams

Cons

  • More initial setup
  • Need shared package for UI
B

One App, Two Portals

apps/admin with middleware-based subdomain routing

Single Next.js app serves both subdomains. /admin/* for admins, /teacher/* for teachers. Middleware checks subdomain + role and redirects.

Pros

  • Single codebase to maintain
  • Shared auth logic
  • One deployment

Cons

  • Slightly more complex routing
  • Admin/teacher code mixed
C

Extend Existing Web App

Add (admin)/ and (teacher)/ route groups to apps/web

All three subdomains served by one app via Next.js middleware.

Pros

  • Minimal setup - one app total
  • Shared fonts, theme, components

Cons

  • Mixes public site with internal tools
  • Bundle size bloat
  • Harder to access-control

My recommendation: B - One App, Two Portals. Simpler than two separate apps, cleaner than mixing with the public site. Admin and teacher share auth, DB access, and UI patterns - they belong together.
